This package contains examples to work with the eSciDoc Java Client library by bypassing the Java class mapping.

All examples of this package calling methods which communicate via REST interface with the framework. The XML representation is used within method calls.

You are supported by XML templates. These templates has to be altered during a example run. Usually you have to replace the place holder (###PLACEHOLDER_NAME###) with a value which fits in.

You find detailed instructions,which values are to alter, inside the classes.

These are code examples to explain the functionality and to give you tool to start with playing with eSciDoc with little constrains.
If you building an productive environment it's recommend to use a HTTP client implementation (e.g. apache httpclient). Using streams is an advantage of an HTTP client.
